Introduction

Python is an interpreted high-level general-purpose programming language. Its design philosophy emphasizes code readability with its use of significant indentation. Its language constructs as well as its object-oriented approach aim to help programmers write clear, logical code for small and large-scale projects.

Python is dynamically-typed and garbage-collected. It supports multiple programming paradigms, including structured (particularly, procedural), object-oriented and functional programming. It is often described as a "batteries included" language due to its comprehensive standard library.

Code example

To get started with Python, open your perfered code editor and insert and run the following code.

Hello world:

print('Hello, world!')

Factorial of a positive integer:

n = int(input('Type a number, and its factorial will be printed: ')) if n < 0:
raise ValueError('You must enter a non-negative integer')
factorial = 1 for i in range(2, n + 1):
factorial *= i
print(factorial)
Variables

Python has no command for declaring a variable. A variable is created the moment you first assign a value to it. Variables do not need to be declared with any particular type, and can even change type after they have been first set.

  • A variable name must start with a letter or the underscore character
  • A variable name cannot start with a number
  • A variable name can only contain alpha-numeric characters and underscores (A-z, 0-9, and _ )
  • Variable names are case-sensitive (age, Age and AGE are three different variables)
x = 4 #x is of type int x = "Sally" #x is now of type str print(x)
Variable scope

A scope defines the visibility of a name within a block. If a local variable is defined in a block, its scope includes that block. If the definition occurs in a function block, the scope extends to any blocks contained within the defining one, unless a contained block introduces a different binding for the name. When a name is used in a code block, it is resolved using the nearest enclosing scope. The set of all such scopes visible to a code block is called the blockâ€™s environment.

The global statement has the same scope as a name binding operation in the same block. If the nearest enclosing scope for a free variable contains a global statement, the free variable is treated as a global.

x = "awesome" def myfunc():
print("Python is " + x)
myfunc() #prints "Python is awesome"

The nonlocal statement causes corresponding names to refer to previously bound variables in the nearest enclosing function scope. SyntaxError is raised at compile time if the given name does not exist in any enclosing function scope.

x = "awesome" def myfunc():
x = "fantastic"
print("Python is " + x)
myfunc() #prints "Python is fantastic"
Data types

Python uses duck typing and has typed objects but untyped variable names. Type constraints are not checked at compile time; rather, operations on an object may fail, signifying that the given object is not of a suitable type. Despite being dynamically-typed, Python is strongly-typed, forbidding operations that are not well-defined (for example, adding a number to a string) rather than silently attempting to make sense of them.

The most common types used in Python are:

  • bool: True and False.
  • dict: Associative array (or dictionary) of key and value pairs; can contain mixed types (keys and values), keys must be a hashable type
  • float: Double-precision floating-point number. The precision is machine-dependent but in practice is generally implemented as a 64-bit IEEE 754 number with 53 bits of precision.
  • int: Integer of unlimited magnitude.
  • list: Also known as an Array in other languages, can contain mixed types.
  • str: A character string
  • tuple: A collection which is ordered and unchangable
Function declorations

In Python a function is defined using the 'def' keyword. To call a function, use the function name followed my parenthesis holding any argumets, if any, to pass through the function.

def my_function():
print('Hello from a function')

By default, a function must be called with the correct number of arguments. Meaning that if your function expects 2 arguments, you have to call the function with 2 arguments, not more, and not less.

def my_function(fname, lname):
print(fname + " " + lname)
my_function("Emil", "Refsnes")
